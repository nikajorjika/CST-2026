An interactive demo of every concept from the slides

1. CSS Syntax

selector {
  property: value;
}

A CSS rule consists of a selector and a declaration block. Each declaration is a property: value pair, ended with a semicolon.

2. Selectors

Wildcard *

Targets every element in the DOM.

The rule * { box-sizing: border-box; margin: 0; padding: 0; } is applied to every single element on this page — including this paragraph.

Element Selector p, span

Targets all elements of a specific HTML tag.

This <p> is styled by the p element selector — light blue background.

This <span> is styled by the span element selector — italic text.

Class Selector .highlight

Targets every element that has a given class. The same class can be applied to any HTML element.

This <p> has class="highlight" — styled.

This <p> has no class — not styled by the class selector.

This <div> also has class="highlight" — styled too!

ID Selector #special-para

Targets exactly one element with a specific id. IDs must be unique per page.

This <p> has id="special-para" — styled by the ID selector.

This <p> has no ID — not styled.

3. Advanced Selectors

Descendant Selector .ancestor p

Targets every <p> that is inside .ancestor, no matter how deeply nested.

Direct child <p> — styled (teal border).

Deeply nested <p> — also styled (still a descendant).

Child Selector .direct-parent > p

Targets only the <p> elements that are a direct child of .direct-parent.

Direct child <p> — styled (orange background).

Nested <p> (not a direct child of .direct-parent) — NOT styled.

Adjacent Sibling Selector h4 + p

Targets the first <p> that immediately follows an <h4>.

Heading (h4)

First <p> directly after h4 — styled (green background).

Second <p> — NOT styled by adjacent sibling selector.

Third <p> — NOT styled by adjacent sibling selector.

General Sibling Selector h4 ~ p

Targets all <p> siblings that come after an <h4>.

Heading (h4)

First <p> after h4 — styled.

Second <p> — also styled!

Third <p> — also styled!

4. Pseudo-classes & Pseudo-elements

User Action: :hover, :active, :focus

These pseudo-classes respond to user interaction. Try hovering, clicking, and tabbing through the elements below.

Structural: :first-child, :last-child, :not(:first-child)

Target elements based on their position among siblings.

  • Item 1 — :first-child → bold + blue background
  • Item 2 — :not(:first-child) → italic
  • Item 3 — :not(:first-child) → italic
  • Item 4 — :last-child + :not(:first-child) → underlined + italic + pink

Form States: :checked, :disabled, :required, :invalid

:disabled — grayed out, not interactive
:required — orange border
:invalid — red border (wrong format)

Pseudo-elements: ::before, ::after, ::placeholder

Pseudo-elements insert generated content or style specific parts of an element.

This paragraph gets content injected by ::before and ::after.

5. Specificity

When multiple CSS rules target the same element and property, specificity determines which rule wins. Higher score = wins.

p, div…

Specificity in action — same element, increasing winner

All four paragraphs below have every rule applied to them, but the one with the highest specificity sets the final color.

p { color: gray } — element selector wins (score: 1)

.spec-class { color: royalblue } — class beats element (score: 10)

#spec-id { color: crimson } — ID beats class (score: 100)

style="color: darkorchid" — inline beats ID (score: 1000)
